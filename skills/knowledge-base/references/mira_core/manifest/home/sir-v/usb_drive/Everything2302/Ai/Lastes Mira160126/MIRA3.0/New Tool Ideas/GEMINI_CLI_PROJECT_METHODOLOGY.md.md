### 7. Ways of Working & Workflows

This methodology promotes a highly collaborative and transparent way of working with the Gemini CLI agent.

Documentation-Driven Development

All significant project information, decisions, and progress are captured directly in the project's markdown files.

The agent (and any human developer) can always find the latest context by reading the files. This eliminates the need for constant verbal updates.

Agent's Role:

The agent actively participates in updating these documents as part of its task completion.

Iterative & Incremental Development

Break down complex features into small, manageable, and testable steps.

Reduces complexity, allows for frequent verification, and makes it easier to recover from errors.

Agent's Role:

The agent proposes and executes these small steps, reporting progress after each one.

Persona-Based Reasoning (for Agents like MIRA)

The agent utilizes distinct cognitive personas (e.g., First Principles, System Architect, Pragmatic Application) to analyze problems from multiple angles.

Leads to more comprehensive, nuanced, and robust solutions.

Agent's Role:

The agent explicitly states which personas it is employing during its reasoning process, making its thought process transparent.

Tool-Assisted Understanding

The agent uses its available tools (e.g.,

list_directory

search_file_content

) to gather information directly from the codebase and project files.

Ensures the agent's understanding is grounded in the actual project state, not just its conversational memory.

Agent's Role:

The agent proactively uses these tools to refresh its context or investigate specific areas of the project.
