# ACCT Semantic Weave Protocol (SWP)

## 1. Objective: Universal Networking
The Semantic Weave Protocol (SWP) establishes the formal rules for creating, linking, and tagging "Zettels" in the ACCT Memory Mesh. It ensures that every new data point is networked into the system's "Second Brain," preventing information from becoming isolated or forgotten.

## 2. The Zettel Structure:
Every new note or documentation file added to the Memory Mesh should include a **Metadata Header**:
*   **UID:** Unique timestamp-based ID (e.g., `20260302-1400`).
*   **Tags:** Semantic keywords for Active Recall (e.g., `#protocol`, `#titan`, `#lab`).
*   **Links:** References to other relevant Zettels (e.g., `[[ACCT_MANIFESTO]]`).
*   **Nodal Context:** Which Cognitive Node created or primarily uses this note.

## 3. Linking & Tagging Rules:
*   **Rule of Three:** Every new Zettel should ideally link to at least **three** existing files in the Memory Mesh to ensure strong structural integration.
*   **Semantic Overlap:** Use tags that are consistent across the ecosystem (e.g., if a file uses `#vision`, also use `#titan` if applicable).
*   **Reference Hierarchy:** High-level protocols (like this one) should be linked to by any specific implementation notes or logs.

## 4. Maintenance (Weave Consolidation):
*   **Bi-Weekly Audit:** ACCT will proactively scan for "Isolated Nodes" (notes with fewer than 2 links) and propose new connections to the Commander.
*   **Contextual Merging:** If two Zettels are found with >80% semantic overlap, ACCT will propose a "Merge Quest" to consolidate the knowledge.
