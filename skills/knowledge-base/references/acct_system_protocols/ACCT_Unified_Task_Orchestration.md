# ACCT Unified Task Orchestration (UTO) Workflow

## Core Workflow: ACCT's Dynamic Cognitive Orchestration System

### Purpose:
The ACCT Unified Task Orchestration (UTO) Workflow is the meta-framework for managing and executing complex, contextual thinking tasks. It replaces static assistance with an intelligent, adaptive execution engine that synchronizes ACCT's Nodal Cognitive Mesh with the user's Memory Mesh.

### Phases of the ACCT UTO Workflow:

1.  **Intent Recognition & Node Mapping**
    *   **Objective:** Precisely parse the user's request and map it to specific Cognitive Nodes (Analytic, Empirical, Creative, etc.).
    *   **Process:**
        *   **Cognitive Analysis:** Identify which Nodes are best suited for the current "Quest."
        *   **Contextual Anchoring:** Search the Memory Mesh for related historical data to ground the task.
        *   **Sub-Goal Decomposition:** Break the objective into a sequence of verifiable tasks in `TODO.md`.

2.  **Memory Mesh Allocation & Synthesis**
    *   **Objective:** Marshal all necessary data from the Second Brain and Skill Vault to prepare for execution.
    *   **Process:**
        *   **Active Recall:** Proactively retrieve and synthesize relevant Zettels or logs.
        *   **Tool Parameterization:** Prepare Skill Vault modules with pre-computed contextual inputs.
        *   **Resource Mapping:** Ensure all external file paths and dependencies are verified.

3.  **Dynamic Logic Chaining & Execution**
    *   **Objective:** Construct and execute a flexible chain of cognitive activations and tool calls.
    *   **Process:**
        *   **Adaptive Flow:** Adjust the reasoning path based on intermediate findings (e.g., switching from Creative to Empirical if data contradicts a hypothesis).
        *   **Execution Shield:** Perform safety and security checks before any system-modifying tool calls.
        *   **Nodal Synthesis:** Combine the outputs of multiple Cognitive Nodes into a coherent reasoning stream.

4.  **Meta-Learning & Memory Enrichment**
    *   **Objective:** Synthesize results, update the Memory Mesh, and perform architectural self-critique.
    *   **Process:**
        *   **The Weave Update:** Add new insights and connections to the Second Brain as permanent Zettels.
        *   **Telemetry Logging:** Record the effectiveness of the chosen execution chain for future UTO optimization.
        *   **Recursive Refinement:** Update core protocols or templates based on the Quest's outcome.

### **Contextual Value:**
The ACCT UTO Workflow ensures that every task is executed with the full depth of the user's historical context, while simultaneously strengthening the system's underlying intelligence for future Quests.
