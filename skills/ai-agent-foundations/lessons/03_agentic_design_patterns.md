# Lesson 03: AI Agentic Design Principles

## Introduction

There are many ways to think about building AI Agentic Systems. Given that ambiguity is a feature and not a bug in Generative AI design, it's sometimes difficult for engineers to figure out where to even start. We have created a set of human-centric UX Design Principles to enable developers to build customer-centric agentic systems to solve their business needs.

In general, agents should:
- Broaden and scale human capacities
- Fill in knowledge gaps
- Facilitate and support collaboration
- Make us better versions of ourselves

## The Agentic Design Principles

### Agent (Space)
This is the environment in which the agent operates.
- **Connecting, not collapsing** – help connect people to other people, events, and actionable knowledge
- **Easily accessible yet occasionally invisible** – agent operates in background and nudges when relevant

### Agent (Time)
How the agent operates over time:
- **Past**: Reflecting on history with context
- **Now**: Nudging more than notifying
- **Future**: Adapting and evolving

### Agent (Core)
- **Embrace uncertainty but establish trust**

## Implementation Guidelines

1. **Transparency**: Inform user that AI is involved
2. **Control**: Enable user to customize and personalize
3. **Consistency**: Aim for multi-modal experiences across devices
