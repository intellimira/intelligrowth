# The Growth Loop

## MIRA's Core Operating Cycle

The **Growth Loop** is MIRA's fundamental operating cycle - a continuous quest-driven process of receiving tasks, executing them, learning from outcomes, and evolving.

## The Five Phases

┌─────────────────────────────────────────────────────────────────┐

### Phase 1: Receive Quest
- User presents a task/goal/request
- The Dark Passenger performs initial pattern recognition
- Quest is categorized and prioritized

### Phase 2: Assess & Plan
- First Principles break down the quest to fundamentals
- Scientific Method creates hypotheses for approach
- UTO Workflow initiates Intent Recognition & Skill Mapping
- Resource allocation strategy is determined

### Phase 3: Execute
- Pragmatic Application drives the execution
- Dynamic Execution Chain activates tools and personas
- Real-time adaptation occurs as needed
- Progress is monitored by Agent Telemetry

### Phase 4: Learn & Adapt
- Scientific Method analyzes outcomes
- Performance telemetry is reviewed
- Success/failure patterns are identified
- Weave is updated with new knowledge

### Phase 5: Synthesize & Report
- Creative Synthesis produces novel insights
- Philosophical Inquiry ensures ethical alignment
- Results are formatted for the user
- Recommendations for future quests are generated

## Solo-Leveling Progression

Each completed Quest contributes to MIRA's progression through the Solo-Leveling ranks:
- **E-Rank:** Basic task execution
- **D-Rank:** Improved efficiency
- **C-Rank:** Strategic planning
- **B-Rank:** Autonomous optimization
- **A-Rank:** Self-correcting operations
- **S-Rank:** Strategic Commander
- **God-Rank:** (Aspirational) Perfect self-optimization

## Quest Examples by Rank

| Rank | Example Quests |
|------|----------------|
| E-D | File operations, simple queries, single tool calls |
| C-B | Multi-step projects, complex analysis, skill integration |
| A-S | Self-optimization, novel problem-solving, strategic planning |

## OpenJarvis Integration

### Learning from Local Inference

The Growth Loop now leverages OpenJarvis trace data for continuous improvement:

1.  **Receive Quest** → Local inference ready
2.  **Assess & Plan** → Persona selection via bridge
3.  **Execute** → OpenJarvis agents + local models
4.  **Learn & Adapt** → Trace analysis from OpenJarvis
5.  **Synthesize** → Weave update with local context

### Trace Collection

OpenJarvis automatically records interaction traces:
- Query patterns
- Model selections
- Latency metrics
- Token usage
- Success/failure outcomes

### Self-Improvement

Using OpenJarvis learning policies:
- **sft**: Supervised fine-tuning from successful traces
- **icl_updater**: In-context learning from patterns
